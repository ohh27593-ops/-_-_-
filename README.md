# -_-_-
OpenDART API를 활용해 최근 공시 데이터를 수집하고, 그중 최대주주 변경 이후 CB 발행이나 유상증자와 같은 흐름이 이어지는 기업을 선별하기 위해 만든 Python 기반 데이터 처리 프로젝트입니다.

단순히 공시를 조회하는 데 그치지 않고,
공시 수집 → 공시 유형 분리 → 조건 필터링 → 결과 저장
까지의 흐름을 직접 구현했습니다.

- 프로젝트 목적: 
최근 불공정거래나 무자본 M&A 관련 사례에서는 지배구조 변화 이후 CB, 유상증자 등의 자금조달 수단이 반복적으로 등장하는 경우가 있습니다.
이 프로젝트는 이러한 흐름을 공시 데이터 기준으로 정리하고,
최대주주 관련 공시 이후 후속 자금조달 공시가 이어진 기업을 선별하는 것을 목표로 합니다.

- 사용 기술: 
Python
pandas
requests
OpenDART API
정규표현식(Regex)

- 직접 구현한 내용: 
OpenDART API를 활용한 공시 수집 로직 구현
최근 3년 공시를 90일 단위로 나누어 수집하는 구조 설계
주요사항보고서와 거래소공시를 나누어 수집하는 코드 작성
정규표현식 기반으로 CB, 유상증자, 최대주주 관련 공시 분류
최대주주 관련 공시 이후 CB 또는 유상증자 공시가 이어진 기업 필터링

- 이 프로젝트는 아래 순서로 동작합니다: 
최근 3년 공시 데이터 수집
주요사항보고서에서 CB / 유상증자 관련 공시 분리
거래소공시에서 최대주주 변경 관련 공시 분리
최대주주 관련 공시 이후 CB 또는 유상증자 공시가 이어졌는지 확인
조건에 맞는 기업 목록을 엑셀로 저장

- 아쉬웠던 점: 
이 프로젝트는 OpenDART API를 활용해 공시 리스트를 수집하고 조건에 맞는 기업을 선별하는 구조로 구현했지만, 몇 가지 한계가 있었습니다.
첫째, API 호출에 걸리는 시간 문제 때문에 조회 범위를 최근 3년으로 제한해야 했습니다. 더 긴 기간을 대상으로 분석하려면 호출량과 처리 시간이 크게 늘어나 현재 방식으로는 비효율적이었습니다.
둘째, 실제 분석에는 공시 본문보다 공시 리스트 데이터를 주로 활용했습니다. API로 받은 데이터 중에서도 프로젝트 목적에 직접 활용할 수 있는 정보는 제한적이었고, 상당수 데이터는 선별 과정에서 사용하지 못했습니다.
셋째, 공시 본문 내용을 직접 활용하지 못한 점이 가장 큰 한계였습니다. 공시 본문은 DART 지침상 형식이 완전히 규격화되어 있지 않아, 단순한 규칙 기반 코드만으로는 필요한 정보를 안정적으로 추출하기 어려웠습니다. 특히 지분 관계나 지분권자 정보처럼 문맥 해석이 필요한 항목은 문자열 처리만으로 다루기 어려웠습니다.

- 확장 방향: 
차후에는 공시 본문까지 분석 범위를 넓히고 싶습니다. 특히 Python으로 수집한 공시 본문을 LLM이 읽고, 그 안에서 지분 관계 및 지분권자 정보를 추출할 수 있다면 지금보다 훨씬 정교한 분석이 가능해집니다.
국내 무자본 M&A와 미공개정보 이용 사례는 단발성 사건이라기보다, 경험 많은 관련 인물들이 여러 시기에 걸쳐 반복적으로 개입하는 경우가 많다고 생각합니다. 따라서 본문 수준의 정보까지 추출할 수 있게 되면, 과거부터 현재까지 국내 무자본 M&A 및 미공개정보이용의 실태를 통시적으로 정리하고, 관련 흐름을 도식화하는 방향으로 확장할 수 있다고 봅니다.
